# White Belt badge template

> **Use this after Boss Fight W-B merges.** The badge is not a reading chapter. It is the builder-facing evidence packet for certification.

Copy the template into the program tracker or the approved evidence location.

```markdown

- setup verification is GREEN;
- sandbox PR evidence exists;
- one genuine typo or equivalent micro-fix was merged;
- no evidence includes sensitive material.
```

---

## What changed for me

Write 3-5 sentences in your own words:

```markdown
Before White Belt, I...
The hardest part was...
The thing that now feels less scary is...
The next small thing I want to try is...
```

This paragraph is not for ranking. It helps the program improve the next cohort.

---

## Recertification

```markdown
Date awarded:
Recertification due:
Refresh note:
```

Set recertification due to one year from the awarded date. See [Appendix L - Certification](../../appendices/L-certification/README.md) for the program rule.

## Next-belt hint

```markdown

```

Most White Belt graduates should start with the Yellow Belt boss-fight example when the module set is still being filled in, then return to the Yellow modules as they land.

---

## Certification checklist

- [ ] W-0 evidence exists and is GREEN.
- [ ] W-1 PR URL exists.
- [ ] W-B PR is merged.
- [ ] Reviewer attestation exists.
- [ ] Recertification due date is set.
- [ ] No sensitive details are present in the badge packet.

When all six are checked, the White Belt claim is ready for the tracker.
